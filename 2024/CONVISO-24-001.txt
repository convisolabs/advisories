[CONVISO-24-001] - LettaAI (MemGPT) - Unauthorized Access to User Data via Admin API Endpoint

1. Advisory Information

   Conviso Advisory ID: CONVISO-24-001
   CVE ID: CVE-2024-39025
   CVSSv3 Score: N/A
   Date: 2024-07-XX

2. Affected Components

   LettaAI (MemGPT)

   * Version 0.3.17

3. Description

   An improper authorization vulnerability [CWE-285] in LettaAI (MemGPT) may allow
   unauthorized users to access sensitive user information through the administrative
   API endpoint "/users".

   The vulnerability exists because the endpoint does not properly enforce access
   control restrictions before returning user information stored in the database.

4. Details

   * The vulnerability affects the administrative API endpoint "/users".
   * The endpoint is responsible for retrieving user records from the application database.
   * Authorization checks are not properly enforced before processing requests.
   * An attacker may remotely invoke the endpoint and obtain information belonging to
     other users.
   * The issue results in unauthorized disclosure of sensitive user information.
   * The vulnerability violates the principle of least privilege by exposing
     administrative functionality to unauthorized actors.

   Vulnerability Type:

   * CWE-285: Improper Authorization

   Affected Endpoint:

   * GET /users

5. Solution

   Upgrade to the latest available version of LettaAI (MemGPT).

   Additionally:

   * Restrict access to administrative endpoints.
   * Implement server-side authorization checks for privileged functions.
   * Enforce role-based access control (RBAC) validation before returning user data.
   * Review all administrative API endpoints for similar authorization flaws.
