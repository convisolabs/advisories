[CONVISO-25-002] - TastyIgniter - Unauthorized Order Status Modification Due to Missing Permission Checks

1. Advisory Information

   Conviso Advisory ID: CONVISO-25-002
   CVE ID: CVE-2024-44314
   CVSSv3 Score: 6.5
   Severity: Medium
   Date: 2025-03-18

2. Affected Components

   TastyIgniter

   Affected Version:

   * 3.7.6 and prior versions

   Affected Component:

   * app/admin/controllers/Orders.php
   * index_onUpdateStatus() function

3. Description

   An Incorrect Access Control vulnerability [CWE-284] in TastyIgniter 3.7.6
   allows unauthorized users to update order statuses due to missing permission
   checks within the Orders Management System.

4. Details

   * The vulnerability exists in the Orders Management System.
   * The affected functionality is implemented in the
     index_onUpdateStatus() function located in Orders.php.
   * The application fails to properly verify whether the current user
     possesses permission to modify order statuses.
   * Authorization checks are missing before processing the update request.
   * An attacker with insufficient privileges may remotely manipulate
     order statuses.
   * Successful exploitation may result in unauthorized modification
     of business transactions and workflow integrity violations.

   Vulnerability Type:

   * CWE-284: Improper Access Control

   Affected Function:

   * index_onUpdateStatus()

   Affected File:

   * app/admin/controllers/Orders.php

5. Solution

   Upgrade TastyIgniter to a version newer than 3.7.6 once an official fix
   becomes available.

   Additionally:

   * Enforce permission validation before processing status updates.
   * Apply role-based access control (RBAC) checks to order management actions.
   * Restrict status modification operations to authorized roles only.
   * Review administrative controllers for similar authorization flaws.
